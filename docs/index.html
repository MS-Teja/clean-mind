The idea

Most of a developer's wasted space is regenerable. Clean Mind knows which — and shows you the ↻ command that rebuilds it.

Why Clean Mind

A cleaner that understands developer machines.

Beyond showing what takes space, it explains why each item is safe — and gives color to meaning, not decoration.

A treemap that makes sense

A squarified, drill-down map with breadcrumbs and tier-colored badges — the biggest space hogs are the biggest tiles.

It knows developer junk

A deterministic rules engine recognizes node_modules, cargo target/, DerivedData, package caches and more across six ecosystems — and tells you how each regenerates.

Safe by construction

A three-tier model, a hard denylist nothing can override, and Trash-first deletion. Permanent delete lives behind a type-to-confirm gate. Nothing is ever deleted automatically.

AI on your terms

Bring your own Anthropic or OpenAI-compatible key, or run a local model via Ollama. It only ever sees directory metadata — and can never promote something to "safe" on its own.

Private & offline by default

No telemetry, no account, no bundled inference. Fresh scan each launch — nothing cached, nothing running in the background.

Native and fast

A parallel Rust core — hardlink-aware and APFS-clone-aware — under a Flutter UI. It walks your home directory and builds a size map in seconds.

The map

Every tile sized by the space it takes.

An interactive treemap turns an abstract number into terrain you can read at a glance. Green tiles are confirmed safe to reclaim; a running total sits in the header.

  • Drill down with breadcrumbs; double-click any tile to zoom in.
  • Tier-colored badges mark what's reclaimable and what's protected.
  • Reveal in Finder, or move straight to Trash.

The reasoning

For each item: why it's safe, and how it comes back.

The insights panel groups reclaimable items by kind, and for every one explains what it is, what deleting it costs you, and the exact command that regenerates it.

  • cargo build, npm install — the rebuild command, spelled out.
  • Optional AI pass sends folder names and sizes — never contents.
  • Select what you trust; nothing is one-click unless a rule verified it.

How it works

Five steps, from full disk to reclaimed space.

01 · SCAN

Scan

A parallel Rust scanner walks your home directory and builds a size map. Fresh every launch.

02 · SEE

Understand

An interactive treemap shows where the space actually went.

03 · SORT

Classify

The rules engine recognizes known artifacts and marks them by how safely they regenerate.

04 · ASK

Ask (optional)

An LLM you configure reviews metadata-only summaries of your largest directories.

05 · CLEAN

Clean

Deletions go to the OS Trash by default and stay recoverable. Permanent delete is gated.

The trust model

The LLM is never trusted on its own.

Everything Clean Mind finds lands in one of three tiers. Color carries meaning here — and protected items always pair with a lock, so meaning is never carried by color alone.

Tier 1

Safe · regenerable

Only items confirmed by the rules engine — like node_modules next to a package.json. Fully reproducible.

One-click reclaim to Trash
Tier 2

Review

LLM suggestions that no rule verifies. Always shown with reasoning — never one-click, and never deleted automatically.

Human confirms every item
Tier 3

Protected

A hard-coded denylist — documents, photos, .ssh, system paths — that neither the rules nor the LLM can override.

Never selectable for deletion

Privacy

AI is opt-in, and kept on a short leash.

The optional analysis pass is the only thing that ever leaves your machine — and it's built so it can't leak your files or overrule the safety model.

  • Metadata onlyPaths, sizes and timestamps — never file contents.
  • Optional redactionPseudonymize path names under your home before anything leaves.
  • Keys in the keychainAPI keys live in your OS keychain, not in config files.
